Deeper dive in to FastRank science

01

Seed Lists

Finding raw domains
The best domains come from the best seed lists, which we generate through our learning algorithm.

To get the best seed lists, we have a categorisation and tagging model, so when a good domain is found we know where it's come from.

Using the learning algorithms from Amazon machine learning, we look for certain characteristics in our seed list domains, which we then aggregate into seed lists for specific types, topics and power of domains were interested in. This might be iGaming, real estate, car rental or whatever.

02

Crawler

Infrastructure and scale
We run crawlers directly from our own servers and because they custom coded, they're very efficient and easy to scale.

Our crawler infrastructure is custom coded to improve its efficiency and controllability. Because our seed lists are pretty complex, were able to give the crawler very detailed instructions, which means it finds more of the kind of domains we want .

We typically crawl around 8 - 12 million domains a day and whilst by some benchmarks thats pretty small, we decided to make our crawling focused and agile because it gives us the best results. Over time, the crawler infrastructure will of course grow.

More: Crawler
03

Mass Whois lookups

Can we register the domain?
As we find error 404 links, we check the root domain with WHOIS API, to see if the domain has expired.

The crawler is looking for broken links which I then checked against the WHOIS database. This database tells us whether a domain is still registered or not. We use an API service from xWhoapi because it's very reliable and we can use it at scale.

Important thing at this stage, is finding enough domains and processing them through the WHOIS API.

More: Whois
04

Domains Found

1st Stage Processing
Once we identify a domain as available for registration, we get data from Majestic link analysis tools to see if it's good enough to process.

Once we've identified a domain that's available, we make an API call to Majestic (GetIndexItemInfo) , where we get the 'headline' metrics for that the main. Were typically looking for domains which have a trust flow of 10 and above and more than 10 referring domains.

Were also looking for domains which have a citation flow no greater than twice the trust flow. We set the criteria relatively low so we have more chance of finding solid intermediate and low level domains.

If the domain does pass this basic test, we then run it through the evaluation process which is pretty complex.

More: Domains Found
05

Domain Evaluation

Heavy analysis
Here we use a proprietary link scoring algorithm to work out how strong the domain is. We also categorise the site using AlchemyAPI.

Domain power
Here, domains are analysed using data from Majestic's API call 'xxxxxxxxxxxxx'. We analyse the top 100 best backlinks going to a referring domain. The Majestic data will exclude things like links in frames, deleted links and no follow links.

The links are processed using a proprietorial algorithm which creates something we call a weighted trust flow: WTF! ;-) The WTF score is based on things like the age of a link, the citation flow/trust flow of the referring domain, the number of referring domains.

Relevance
At this point, we also scrape archive.org for previous iterations of the site in question. We take these scrapes and process them through alchemy API, which is a natural language processing tool. It categorises the content very accurately and gives us most prominent keywords on page.

Spam
We also check for spam on domains. We scrape archive.org pages, processing the content of these pages and looking for certain stop words. We also make a call to Majestic API asking for the top 10 anchor texts for each domain. The anchor text is also checked for spam keywords. Any domain which goes above a certain score is marked as spam and dumped.

More: Domain Evaluation
